1. New Family — Application through Payment
VEX V5 and Combat Robotics. Covers application, financial aid gate, registration, and payment confirmation.
▶
Family learns about Placer Robotics
Family
Submits application at placerrobotics.org/apply — no account required. Student info, program interest, long-form motivation answers, guardian contacts.
System
Creates family, guardian, and student records. Sends confirmation email to both guardians. Application appears in admin queue.
Admin
Reviews application (cannot see financial information). Reads long-form answers and GPA. Makes acceptance decision. If applicant selected both programs or Not Sure, admin confirms final program at this step.
Decision
Application accepted or declined?
✗ Declined
Decline notification sent. Workflow ends for this application.
✓ Accepted
Acceptance email sent with login link. Family directed to financial aid prompt.
Family
Logs in via magic link. Proceeds directly to registration form — no intermediate gate.
Note
Financial aid is self-service, never a gate. The registration form includes a callout: "Need financial assistance? Submit a request here — before or after registration." Families who request aid can continue registering immediately. Financial Aid Admin resolves the request and admin adjusts enrollment records if needed. Registration is never locked for financial aid.
Family
Completes multi-section registration form: participant info, program selection, fees and fundraising, volunteer interest, expectations acknowledgment, waivers.
System
Creates enrollment record. Records waiver signatures with legal evidence (typed name, IP, timestamp, waiver version). Generates payment reference code. Triggers Google Group sync.
Family
Sees confirmation page with: Zeffy payment link, payment reference code, fundraising target, deadline, offline payment instructions.
Family
Makes payment via Zeffy, check, Benevity, or corporate match using their payment reference code.
System
Receives Zeffy webhook. Matches payment to enrollment. Updates fee/fundraising status. Sends payment confirmation email to family.
✓
Family enrolled and payment confirmed
2. Volunteer Clearance — 5-Step Process
All programs. Same workflow for V5 coaches, IQ coaches, Combat managers, and general cleared volunteers.
▶
Guardian initiates clearance from family dashboard
Volunteer
Step 1 — Policy Acknowledgment (required for insurance)Volunteer reviews current season youth protection policy, lab use policy, and volunteer conduct policy. Types name as electronic acknowledgment with full evidence capture (version, hash, IP, timestamp). Required written confirmation for insurance. For parents already in the system, no new guardian record is created.
System
Creates volunteer profile. Opens all 5 clearance steps with status = pending.
Admin
Step 1 — CA DOJ Background CheckVolunteer completes Live Scan fingerprinting through external provider. Admin marks Step 1 complete in dashboard after DOJ result confirmed. Date and notes recorded. No API — manual step.
System
Step 2 — APS Youth Protection Training (starts automatically)System calls APS API: creates user, assigns training, retrieves training link. Sends training link to volunteer by email.
Volunteer
Completes ~90 min APS youth protection training on the APS/MinistrySafe website (outside this platform).
System
APS sends completion webhook. System stores certificate URL, issue date, and expiration date. Step 2 marked complete automatically.
Volunteer
Step 3 — Youth Protection Policy QuizTakes in-platform quiz. Must score 90% or higher. Unlimited retakes allowed.
System
Records all attempts. On passing attempt, Step 3 marked complete. Volunteer notified.
Volunteer
Step 4 — Lab Use Policy QuizTakes in-platform quiz. Must score 90% or higher. Unlimited retakes.
System
On passing attempt, Step 4 marked complete.
Admin
Step 5 — Lab OrientationVolunteer completes in-person lab orientation at the Placer Robotics facility. Admin marks Step 5 complete in dashboard. Date recorded.
System
Auto-Clearance — all steps completeVolunteer status → Cleared. Clearance notification sent. Slack workspace invite sent. Google Group sync triggered. UniFi door credentials provisioned via API.
✓
Volunteer cleared — full access granted

3. Annual Volunteer Renewal
Standard: renewal opens July 1, target complete August 31 (both configurable in season_config). All cleared volunteers. Live Scan not repeated on standard renewal.
▶
July 1 (standard) — renewal opens per season_config.volunteer_renewal_open_at
System
Sends renewal invitation email to all cleared volunteers. Renewal link in volunteer dashboard.
Volunteer
Step 1 — Policy Re-Acknowledgment (always required)Reviews current season youth protection policy in full. Types name as electronic acknowledgment. Checks consent boxes.
Decision
Has the youth protection policy changed since this volunteer's last acknowledgment?
Yes — policy changed
Volunteer must retake Youth Protection Quiz (90%+ required, unlimited retakes).
No — unchanged
Quiz step skipped automatically.
Decision
Does this volunteer's APS certificate expire on or before May 31 of this program year?
Yes — cert expiring
System auto-enrolls in new APS training. Training link sent by email. Volunteer completes training on APS site. Completion webhook marks step done.
No — cert still valid
APS step skipped automatically.
System
All required steps complete. Volunteer status remains Cleared. Renewal confirmation sent. Season renewal date recorded.
✓
Renewal complete — cleared for new season

If Renewal Is Not Completed
ScenarioWhat Happens
Steps incomplete, season still openAdmin notified. Volunteer may continue operating. Admin tracks outstanding renewal.
APS cert expires without renewalVolunteer status auto-expires. Removed from Google Groups and Slack on next sync.
Reinstatement after expiryVolunteer completes new APS training. APS webhook auto-reinstates status to Cleared.
4. VEX IQ Coach — Team Creation through Student Registration
IQ coach self-creates team, pays team fee, admin confirms, IQ families register and attach to team.
▶
Coach wants to register an IQ team
IQ Coach
Submits team creation form at placerrobotics.org/iq-team/create — no login required. Enters school name, division, contact info, estimated team size. No team number field — assigned later by admin.
System
Creates team record (status: Pending Payment). Generates team payment reference code. Sends Zeffy payment link and instructions to coach.
IQ Coach
Pays $1,200 team fee via Zeffy using the payment reference code.
System
Receives Zeffy payment webhook. Matches to team. Team fee status → Paid. Team status → Pending Admin Confirmation. Payment Admin notified.
Admin
IQ Coordinator reviews team submission and payment. Confirms team. Team status → Active.
System
Sends coach login link and clearance instructions. Sends magic link registration invitations to each parent/guardian email provided in student roster.
IQ Coach
Logs in and begins volunteer clearance process. (See Diagram 2 — Volunteer Clearance.) Team is active during clearance process. Students may register while coach is pending clearance.
Admin
Marks events.vex.com registration complete on team record once IQ team is registered externally. Also assigns team number when GRSF/VEX confirms it.
IQ Family
Applies, gets accepted, goes through financial aid gate, then reaches IQ registration form. Selects their team from the list of active confirmed IQ teams.
System
Creates enrollment with team assigned. Registration fee = $0. Fundraising target = $0 default. Optional donation Zeffy link displayed.
✓
IQ team active, students registered, coach clearing

7. "Not Sure" Applicant Triage
For students who select "Not Sure" on the application. Triage happens through engagement events before acceptance.
▶
Student applies and selects "Not Sure" for program
System
Application created with program_interest = not_sure. Routes to "Not Sure / Program Pending" sub-queue in admin dashboard. Confirmation email sent to family.
Admin
Reviews application. Reads motivation answers and background. Marks status = Program Pending. Adds triage notes about next steps for this student.
Student
Attends one or more engagement events — info nights, try-it-out sessions, open labs, summer camps, RISE, Girl Powered. Admin optionally records attendance in the system.
Admin
Follows up with student and family. Gathers input from coaches and student directors. Updates triage notes as engagement progresses.
Decision
Has the student identified a program?
Yes — program determined
Admin sets final program to VEX V5, Combat, or both programs. If both selected, system creates one enrollment per program at registration. Updates triage notes. Moves to Accepted.
No — still undecided
Application stays in Program Pending. Continue engagement events. No deadline — admin judgment.
System
Sends acceptance email with login link. Directs family to financial aid prompt. Normal registration flow proceeds.
✓
Student accepted with confirmed program — normal registration flow begins
8. Google Group / Slack Sync Cycle
Replaces the sync_script.py spreadsheet-based sync. Supabase is the source of truth. Runs every 15 minutes and on key events.
▶
Trigger: scheduled (every 15 min) or on-demand after registration, clearance, or role change
System
For each auto-synced Google Group: Queries Supabase for current desired membership based on group rule (enrollment-driven, clearance-driven, or role-driven).
System
Resolves email for each member: uses google_email if provided, otherwise falls back to login_email. No external mapping spreadsheet consulted.
System
Fetches current Google Group membership via Google Admin SDK. Computes diff: who needs to be added, who needs to be removed.
System
Executes batch add/remove via Google Admin SDK. Logs every change to sync_log: group, action, email, which field resolved from, timestamp, success/failure.
System
Slack user group sync: Updates cleared-parents Slack user group in both main and IQ workspaces to match all volunteers with status = cleared.
Decision
Any sync failures (API error, permission issue, unresolvable email)?
Yes — failures
Failures logged to sync_log. Surfaced in admin notification dashboard. Admin may investigate and trigger manual resync.
No — clean
Sync complete. All changes logged.

Google Groups and Slack groups match current Supabase state

9. Returning Family — Six Paths
All returning families use the same account. Financial aid and waivers always reset. Team assignment defaults to prior season.
What Always Resets Each Season
ItemResets?What Family Must DoNotes
Financial aidYes — alwaysRe-apply before registrationNo prior approval carries forward
WaiversYes — alwaysSign new waivers at registrationNo exceptions
Student communication consentYes — alwaysRe-certify at registrationRe-collected each season
Team assignment (V5/Combat)No — defaults to prior teamNothing — admin updates if neededAdmin changes post-registration
Guardian/student recordsNo — pre-filledReview and update at registrationAll fields editable
Volunteer clearanceNo — carries forwardComplete annual renewalUnless expired or suspended

Six Returning Paths
PathWhat's DifferentFamily Action
1. Same program, same teamNothing — most common caseApply (or admin waiver) → financial aid gate → register. Team pre-filled from prior season.
2. Same program, different teamTeam assignment onlySame as Path 1. Admin updates team assignment after registration. Family does nothing different.

e.g. V5 → V5 + Combat		Program selection, fundraising target changes to both-program amount ($1,100)Submit new application indicating both programs. Financial aid gate re-runs. Registration reflects updated program.

e.g. V5 → Combat only		Full program change. Prior enrollment preserved in history.Submit new application with new program. Financial aid gate re-runs. New enrollment created.
5. Add a siblingNew student record under existing familyLog in → initiate new application for sibling from family dashboard. Standard queue. Financial aid for new student handled separately.
6. IQ → V5 transitionFull program change. IQ history visible to admin during review.Submit full V5 application. Admin may use admin_waived to fast-track. V5 fees and targets apply — IQ's $0 fee does not carry forward.
